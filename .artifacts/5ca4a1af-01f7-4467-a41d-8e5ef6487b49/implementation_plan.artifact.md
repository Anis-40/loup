# خطة إضافة نظام "التحديثات التلقائية" (Update Checker)

بما أننا سنرفع اللعبة على GitHub وسيقوم الأصدقاء بتحميلها خارج المتجر، فمن الضروري وجود نظام يخبرهم بوجود نسخة جديدة (مثلاً عند إضافة أدوار جديدة أو إصلاح أخطاء) لضمان أن الجميع يلعب بنفس النسخة.

## آلية العمل المقترحة

سنعتمد على نظام "التنبيه بالتحديث" (Update Notification):
1.  سنضع ملفاً صغيراً جداً على GitHub يسمى `version.json` يحتوي على رقم أحدث نسخة ورابط التحميل.
2.  عند فتح اللعبة، سيقوم التطبيق "بسرية" بالتأكد من هذا الملف عبر الإنترنت.
3.  إذا وجد نسخة أحدث من النسخة المثبتة على الهاتف، ستظهر نافذة منبثقة (Popup) جميلة تخبر اللاعب: "هناك تحديث جديد متاح! هل تريد التحميل؟".
4.  عند الضغط على "تحديث"، سيتم توجيهه مباشرة لرابط تحميل ملف الـ APK الجديد أو صفحة GitHub.

## التغييرات المطلوبة

### 1. إضافة المكتبات اللازمة
- إضافة `http` لإجراء عملية الفحص عبر الإنترنت.
- إضافة `package_info_plus` لقراءة رقم النسخة الحالية من داخل التطبيق تلقائياً.

### 2. خدمة التحديث (Update Service)
- **[NEW] [update_service.dart](file:///C:/Users/admine/.gemini/antigravity/brain/8ececb92-2064-43f3-84d4-310c7f81fe51/loup/lib/core/services/update_service.dart)**:
    - وظيفة لجلب البيانات من GitHub.
    - مقارنة رقم النسخة الحالية بالرقم الموجود في السيرفر.

### 3. واجهة التنبيه (UI)
- تصميم نافذة منبثقة (Dialog) تتوافق مع جمالية اللعبة، تحتوي على:
    - رقم النسخة الجديدة.
    - قائمة بالمميزات الجديدة (What's new).
    - زر "تحديث الآن".

### 4. الربط مع بداية التشغيل
- استدعاء فحص التحديث فور تشغيل التطبيق في الملف الرئيسي `main.dart`.

## خطة التحقق (Verification)

1. محاكاة وجود نسخة قديمة في الكود ونسخة جديدة في ملف الـ JSON (سنستخدم رابطاً تجريبياً).
2. التأكد من ظهور نافذة التحديث فور فتح اللعبة.
3. التأكد من أن زر "تحديث" يفتح الرابط الصحيح في المتصفح.

> [!IMPORTANT]
> **ملاحظة تقنية**: نظام آيفون (iOS) لا يسمح بالتحديث التلقائي الفعلي خارج المتجر، لذا سيقوم التطبيق بتنبيه صديقك بوجود نسخة جديدة، وعليه هو إعادة استخدام Sideloadly لتثبيت النسخة الجديدة (هذه قيود آبل الصارمة). أما في أندرويد فالأمر أسهل بكثير.
